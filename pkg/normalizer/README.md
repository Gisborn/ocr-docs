# pkg/normalizer

Нормализатор данных паспорта РФ — парсит сырой текст от OCR в структурированные поля.

## Назначение

Преобразует распознанный текст:
```
ИВАНОВ ИВАН ИВАНОВИЧ
01.01.1990
СЕРИЯ 4515 НОМЕР 123456
ВЫДАН 15.05.2015
ОТДЕЛОМ УФМС РОССИИ
КОД 770-064
```

В структурированный JSON:
```json
{
  "last_name": "Иванов",
  "first_name": "Иван",
  "middle_name": "Иванович",
  "birth_date": "01.01.1990",
  "series": "4515",
  "number": "123456",
  "issue_date": "15.05.2015",
  "issued_by": "ОТДЕЛОМ УФМС РОССИИ",
  "division_code": "770-064"
}
```

## Использование

```go
n := normalizer.New()

result, err := n.Normalize(rawOcrText)
if err != nil {
    log.Fatal(err)
}

fmt.Println(result.Data.LastName)    // "Иванов"
fmt.Println(result.Confidences.LastName)  // 0.9
```

## Возможности

- **ФИО**: Извлечение фамилии, имени, отчества
- **Даты**: Форматирование в `ДД.ММ.ГГГГ`
- **Серия/номер**: Распознавание форматов `XX XX XXXXXX` и `XXXXXXXXXX`
- **Код подразделения**: Формат `XXX-XXX`
- **Кем выдан**: Извлечение текста после ключевых слов
- **Confidence**: Оценка уверенности по каждому полю

## Тестирование

```bash
go test ./pkg/normalizer/... -v
```

## Требования к точности

Согласно плану: ≥95% на качественных фото паспортов нового образца.

### Ограничения MVP

- Поддержка только паспортов РФ нового образца (с 1997 года)
- Прописка (registration_address) — не распознается, всегда `null`
- Двойные фамилии могут распознаваться некорректно
- Рукописный текст — не поддерживается
