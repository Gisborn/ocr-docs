🔑 Аутентификация

Все запросы к API требуют заголовка X-Api-Key. Ключ создаётся в личном кабинете и показывается только один раз.

Формат ключа: base64(key_id:secret)
Пример: MToxMjM0NTY3ODkwYWJjZGVm

📝 Распознавание паспорта

POST /v1/recognize

Отправляет изображение паспорта на распознавание. Списание средств происходит автоматически.

Заголовки

ЗаголовокОбязательныйОписание
X-Api-KeyДаВаш API-ключ
Idempotency-KeyНетУникальный ключ запроса (UUID). При повторной отправке в течение 24ч вернётся тот же результат без повторного списания.
Content-TypeДаmultipart/form-data

Тело запроса

Content-Type: multipart/form-data

file: <binary image data>  (JPEG, PNG, WebP, макс. 10 МБ)

Пример запроса (cURL)

curl -X POST https://api.adocs.ru/v1/recognize \
  -H "X-Api-Key: ВАШ_API_КЛЮЧ" \
  -H "Idempotency-Key: $(uuidgen)" \
  -F "file=@passport.jpg"

Успешный ответ (200 OK)

{
  "request_id": "req_1751234567890123456",
  "last_name": "Иванов",
  "first_name": "Иван",
  "middle_name": "Иванович",
  "birth_date": "15.03.1990",
  "series": "4515",
  "number": "123456",
  "issue_date": "20.05.2015",
  "issued_by": "ОТДЕЛОМ УФМС РОССИИ ПО Г. МОСКВЕ ПО РАЙОНУ ...",
  "division_code": "770-064",
  "registration_address": null
}

GET /v1/events

Пример запроса

curl -X GET https://api.adocs.ru/v1/events \

📋 Коды ответов

200 OK
Успешное выполнение
401 Unauthorized
Отсутствует или невалидный X-Api-Key
402 Payment Required
Недостаточно средств на балансе
413 Payload Too Large
Файл больше 10 МБ
429 Too Many Requests
Превышен лимит 10 RPS на ключ

⚡ Quick Start

  1. Зарегистрируйтесь в личном кабинете
  2. Создайте API-ключ (раздел «Ключи доступа»)
  3. Пополните баланс через мок-платёж или ЮКассу
  4. Отправьте POST /v1/recognize с изображением паспорта
  5. Получите JSON с распознанными полями
