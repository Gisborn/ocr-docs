> Статус: MVP (реализован, протестирован)  
> Язык документации: Русский
> Последнее обновление: 2026-05-20

## Обзор проекта

**1C-Scan** — облачный API-сервис для распознавания паспортов граждан РФ, предназначенный для интеграции с системой 1С. Сервис позволяет B2B-клиентам автоматически извлекать структурированные данные из фотографий паспортов, исключая ручной ввод.

### Миссия
Исключить ручной ввод паспортных данных в B2B-секторе за счёт облачного OCR-распознавания и бесшовной интеграции с 1С.

### Целевая аудитория
- Абонентские отделы (провайдеры связи, ЖКХ)
- Финансовый сектор (МФО, ломбарды, кредитные кооперативы)
- HoReCa и услуги (отели, турфирмы, аренда авто/оборудования)
- HR-департаменты с массовым наймом
- Аутсорсинговые бухгалтерские и юридические компании

## Архитектура системы

```
1С Расширение / Клиент
     │  POST /v1/recognize  (HTTPS + X-Api-Key)
     ▼
API Gateway  ──── 402 Payment Required (если баланс = 0)
     │
     ▼
Billing Service: проверка баланса → резервирование (PENDING)
     │
     ▼
Idempotency Cache (Redis)
     │  cache hit → ROLLBACK, вернуть кэшированный результат
     │  cache miss → продолжить
     ▼
Yandex Message Queue  ←── буфер, выравнивание пиков
     │
     ▼
Core Orchestrator (Serverless Container)
     ├── OCR: Yandex Vision v2 (primary)
     │         ├── passport model → structured entities
     │         ├── недостаточно полей → generic page model + парсинг
     │         └── fallback → VK Vision v1 (generic OCR)
     └── Нормализатор: ФИО, даты, серия/номер, код подразделения
     │
     ▼
Billing: фиксация результата
     ├── Успех → COMMITTED (списание с баланса)
     └── Тех. ошибка → ROLLBACK (возврат резерва)
     │
     ▼
JSON-ответ → Клиент
```

### Основные компоненты

#### API Gateway
- TLS-терминация
- Аутентификация по заголовку `X-Api-Key`
- Rate-limiting (10 RPS на ключ)
- Маршрутизация по версиям API (`/v1/`, `/v2/`)

#### Billing Service
Двухфазная модель транзакций:
1. **Резервирование (PENDING):** проверка баланса, блокировка суммы
2. **Фиксация:**
   - `COMMITTED` — успешное распознавание, списание средств
   - `ROLLBACK` — техническая ошибка, возврат резерва

Приоритет списания: предоплаченные операции → рублёвый баланс.

#### Core Orchestrator
- Последовательность: OCR → нормализация → ответ
- OCR: Yandex Vision v2 с моделями (`passport`, `driver-license-front` и др.)
  - Structured model возвращает готовые entities без ручного парсинга
  - **Ограничение:** structured model НЕ возвращает confidence по полям — используется дефолтное значение 0.90
  - Fallback на generic `page` модель + текстовый парсинг при неполных полях (там confidence есть на уровне блоков)
  - Fallback на VK Vision v1 при 5xx / таймауте
- Stateless: изображение только в RAM, на диск не пишется
- В v1 вызывается напрямую (синхронно), Yandex Message Queue используется только в планируемой v2

#### Нормализатор
- Разбивка ФИО на отдельные поля
- Форматирование дат: `ДД.ММ.ГГГГ`
- Форматирование серии: `XXXX`, номера: `XXXXXX`
- Форматирование кода подразделения: `XXX-XXX`

## API

### Текущая версия: v1 (синхронная)

```http
POST /v1/recognize
Content-Type: multipart/form-data
X-Api-Key: <api_key>

file: <image_file>
```

**Формат ответа:**
```json
{
  "request_id": "req_01J...",
  "last_name": "Иванов",
  "first_name": "Иван",
  "middle_name": "Иванович",
  "birth_date": "01.01.1990",
  "series": "4515",
  "number": "123456",
  "issue_date": "15.05.2015",
  "issued_by": "Отделом УФМС...",
  "division_code": "770-064",
  "registration_address": null
}
```

### Коды ответов
- `200 OK` — успешное распознавание
- `401 Unauthorized` — отсутствует или невалидный API-ключ
- `402 Payment Required` — недостаточно средств на балансе
- `413 Request Entity Too Large` — файл > 10 МБ
- `415 Unsupported Media Type` — неподдерживаемый формат
- `429 Too Many Requests` — превышен rate limit

### Будущая версия: v2 (асинхронная)
Запланирована после MVP:
- `POST /v2/recognize` — отправка задачи
- `GET /v2/recognize/{request_id}` — polling статуса
- Вебхуки по завершении обработки

---

## Требования к безопасности

1. **Stateless processing:** Изображения не сохраняются на диск, не передаются в хранилища
2. **Шифрование:** TLS 1.2+ для всех соединений, шифрование БД средствами Yandex Cloud
3. **API-ключи:** Хранение в виде bcrypt-хэша, передача только через `X-Api-Key` по HTTPS
4. **Соответствие 152-ФЗ:** Клиент — оператор ПДн, сервис — обработчик
5. **DPA:** Требуется подписание соглашения с Яндексом для использования Yandex Vision
