## Обновление документации

При изменении архитектуры или плана:

1. Обновите соответствующий `.md` файл
2. При изменении API — обновите раздел 4.3 в архитектуре
3. При выполнении задачи из плана — отметьте в плане (добавьте `[x]` или статус "Готово")
4. Синхронизируйте перекрёстные ссылки между документами

## Связь с кодом

- Каждый сервис в `services/` имеет свой `README.md` с деталями реализации
- Интерфейсы в `pkg/` соответствуют абстракциям из раздела 4.2 архитектуры
- Миграции в `migrations/` соответствуют схеме из раздела архитектуры
